miniDashy Build Guide

Player anomaly detection dashboard — Local Docker + Render/Aiven production. Battle-tested and corrected from a real build.

This guide is the corrected version

It fixes four real issues found during an actual build: the broken Step 7 checkpoint that left db not running, the missing <?php tag in db.php, the NgModule vs standalone mismatch for Angular 17, and the missing global CSS that caused a white background with unreadable dark text. All component code is written for standalone Angular 17 — the actual default when you run ng new.

6 nginx/default.conf

How routing works

Nginx is the single entry point on port 8080. Requests to /api/* are forwarded to PHP-FPM via FastCGI on port 9000. Everything else proxies to Angular's dev server on port 4200. The /ng-cli-ws block keeps Angular's hot-reload WebSocket alive through the proxy.

server {
    listen 80;

    location /api/ {
        root /var/www;
        try_files $uri =404;
        fastcgi_pass app:9000;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }

    location / {
        proxy_pass http://frontend:4200;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_read_timeout 300;
        proxy_connect_timeout 300;
        proxy_cache_bypass $http_upgrade;
    }

    location /ng-cli-ws {
        proxy_pass http://frontend:4200;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
    }
}

8 sql/schema.sql

Two tables

players — static identity (name, account age). events — one row per session with behavioral stats. The FOREIGN KEY enforces integrity: you can't insert an event for a non-existent player. utf8mb4 is required — standard MySQL utf8 is 3-byte and can't store emoji.

SET NAMES utf8mb4;
SET CHARACTER SET utf8mb4;

CREATE TABLE IF NOT EXISTS players (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    account_age_days INT NOT NULL
) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS events (
    id INT AUTO_INCREMENT PRIMARY KEY,
    player_id INT NOT NULL,
    kills INT NOT NULL,
    deaths INT NOT NULL,
    accuracy DECIMAL(5,2) NOT NULL,
    session_minutes INT NOT NULL,
    recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (player_id) REFERENCES players(id),
    INDEX idx_player_id (player_id)
) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

10 api/db.php connection + 3 detection functions

⚠ The file MUST start with <?php on line 1

If anything appears before <?php — including a comment — PHP won't execute the file and will serve it as raw text. Angular will receive comment text instead of JSON and fail to parse it with a confusing error. Line 1 must be exactly <?php and nothing else.

Not an endpoint

This file never outputs anything on its own. players.php and player.php load it with require_once. It provides the $pdo connection and three functions. Works unmodified in both local Docker and Render/Aiven — reads everything from environment variables.

<?php
// api/db.php — DB connection + detection functions. NOT an endpoint.
// Loaded via require_once by players.php and player.php.

$host   = getenv('MYSQL_HOST') ?: 'db';
$port   = getenv('MYSQL_PORT') ?: '3306';
$dbname = getenv('MYSQL_DATABASE');
$user   = getenv('MYSQL_USER');
$pass   = getenv('MYSQL_PASSWORD');
$useSSL = getenv('MYSQL_SSL') === 'true';

$dsn = "mysql:host=$host;port=$port;dbname=$dbname;charset=utf8mb4";
$options = [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION];

// PHP 8.5 renamed the SSL constant — check at runtime for whichever exists.
if ($useSSL) {
    $sslConstant = defined('Pdo\Mysql::ATTR_SSL_VERIFY_SERVER_CERT')
        ? Pdo\Mysql::ATTR_SSL_VERIFY_SERVER_CERT
        : PDO::MYSQL_ATTR_SSL_VERIFY_SERVER_CERT;
    $options[$sslConstant] = false;
}

try {
    $pdo = new PDO($dsn, $user, $pass, $options);
} catch (PDOException $e) {
    header('Content-Type: application/json');
    http_response_code(500);
    echo json_encode(['error' => 'Database connection failed: ' . $e->getMessage()]);
    exit;
}

// Rule 1: Fixed threshold.
// Catches: accuracy >80% on account under 14 days old.
// Strength: instant, auditable. Weakness: easy to evade by staying under 80%.
function checkThresholdRule(array $event): ?string
{
    if ($event['accuracy'] > 80.0 && $event['account_age_days'] < 14) {
        return "Accuracy {$event['accuracy']}% on an account only "
             . "{$event['account_age_days']} days old (threshold: >80% under 14 days)";
    }
    return null;
}

// Rule 2: Population-relative z-score.
// Catches: kills/min more than 3 standard deviations above the population mean.
// Strength: adapts to the whole player base, harder to game.
// Weakness: recomputes population stats on every call (fine at this scale).
function checkStatisticalRule(PDO $pdo, array $event): ?string
{
    $stmt = $pdo->query("SELECT kills, session_minutes FROM events");
    $rows = $stmt->fetchAll(PDO::FETCH_ASSOC);

    $rates = array_map(
        fn($r) => $r['session_minutes'] > 0 ? $r['kills'] / $r['session_minutes'] : 0,
        $rows
    );

    $mean = array_sum($rates) / count($rates);
    $variance = array_sum(array_map(fn($r) => ($r - $mean) ** 2, $rates)) / count($rates);
    $stddev = sqrt($variance);

    $playerRate = $event['session_minutes'] > 0
        ? $event['kills'] / $event['session_minutes'] : 0;

    if ($stddev == 0) return null;

    $zScore = ($playerRate - $mean) / $stddev;

    if ($zScore > 3.0) {
        return sprintf(
            "Kill rate %.2f kills/min is %.1f standard deviations above the population mean (%.2f)",
            $playerRate, $zScore, $mean
        );
    }
    return null;
}

// Orchestrator: runs both rules, returns array of fired flags.
// Empty array = clean player.
function detectAnomalies(PDO $pdo, array $event): array
{
    $flags = array_filter([
        checkThresholdRule($event),
        checkStatisticalRule($pdo, $event),
    ]);
    return array_values($flags);
}
11 api/players.php GET /api/players.php

What this returns

All 26 players with their most recent event and flag status. The subquery WHERE e.id = (SELECT MAX(id)...) picks only the latest event row per player. The (int) and (float) casts are required — PDO returns everything as strings by default, which would break Angular's number comparisons.

<?php
// GET /api/players.php — all players with latest event + flag status.

header('Content-Type: application/json');
header('Access-Control-Allow-Origin: *');
header('Access-Control-Allow-Methods: GET, OPTIONS');
header('Access-Control-Allow-Headers: Content-Type');

if ($_SERVER['REQUEST_METHOD'] === 'OPTIONS') { http_response_code(200); exit; }
if ($_SERVER['REQUEST_METHOD'] !== 'GET') {
    http_response_code(405);
    echo json_encode(['error' => 'Method not allowed. Use GET.']);
    exit;
}

require_once __DIR__ . '/db.php';

$stmt = $pdo->query(
    "SELECT p.id, p.name, p.account_age_days,
            e.kills, e.deaths, e.accuracy, e.session_minutes
     FROM players p
     JOIN events e ON e.player_id = p.id
     WHERE e.id = (SELECT MAX(id) FROM events WHERE player_id = p.id)
     ORDER BY p.id"
);
$rows = $stmt->fetchAll(PDO::FETCH_ASSOC);

$results = array_map(function ($row) use ($pdo) {
    $flags = detectAnomalies($pdo, $row);
    return [
        'id'               => (int) $row['id'],
        'name'             => $row['name'],
        'account_age_days' => (int) $row['account_age_days'],
        'kills'            => (int) $row['kills'],
        'deaths'           => (int) $row['deaths'],
        'accuracy'         => (float) $row['accuracy'],
        'session_minutes'  => (int) $row['session_minutes'],
        'flagged'          => count($flags) > 0,
        'flags'            => $flags,
    ];
}, $rows);

echo json_encode($results);
12 api/player.php GET /api/player.php?id=N

Takes real user input — validate immediately

Unlike players.php, this takes a ?id= query parameter. Cast to (int) immediately and reject anything that isn't a positive number. 400 = malformed input, 404 = valid id that doesn't exist.

<?php
// GET /api/player.php?id={id} — one player + full event history + flags.

header('Content-Type: application/json');
header('Access-Control-Allow-Origin: *');
header('Access-Control-Allow-Methods: GET, OPTIONS');
header('Access-Control-Allow-Headers: Content-Type');

if ($_SERVER['REQUEST_METHOD'] === 'OPTIONS') { http_response_code(200); exit; }

require_once __DIR__ . '/db.php';

$playerId = (int) ($_GET['id'] ?? 0);
if ($playerId <= 0) {
    http_response_code(400);
    echo json_encode(['error' => 'Missing or invalid id parameter']);
    exit;
}

$stmt = $pdo->prepare("SELECT id, name, account_age_days FROM players WHERE id = :id");
$stmt->execute(['id' => $playerId]);
$player = $stmt->fetch(PDO::FETCH_ASSOC);

if (!$player) {
    http_response_code(404);
    echo json_encode(['error' => 'Player not found']);
    exit;
}

$stmt = $pdo->prepare(
    "SELECT kills, deaths, accuracy, session_minutes, recorded_at
     FROM events WHERE player_id = :id ORDER BY recorded_at DESC"
);
$stmt->execute(['id' => $playerId]);
$events = $stmt->fetchAll(PDO::FETCH_ASSOC);

$latestEvent = array_merge($events[0] ?? [], ['account_age_days' => $player['account_age_days']]);
$flags = $events ? detectAnomalies($pdo, $latestEvent) : [];

echo json_encode(['player' => $player, 'events' => $events, 'flags' => $flags]);

15 frontend/src/app/app.config.ts required for HttpClient

Do this before writing any components

Without provideHttpClient() here, Angular has no HTTP provider and every API call will fail with NullInjectorError: No provider for HttpClient. This is the standalone equivalent of adding HttpClientModule to app.module.ts in NgModule apps.

import { ApplicationConfig } from '@angular/core';
import { provideHttpClient } from '@angular/common/http';

export const appConfig: ApplicationConfig = {
  providers: [provideHttpClient()]
};

Why npm install first

--skip-install was used during scaffold so Docker handles installs inside the container. But you need node_modules on your host machine too to run ng generate. The npm warn deprecated messages are fine — they're warnings about Angular's own dependencies, not your code.

Then replace the generated file content with:

import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

// providedIn: 'root' = one singleton shared across the whole app.
@Injectable({ providedIn: 'root' })
export class PlayerService {

  // Port 4200 = Angular dev server directly → needs full URL to Nginx on :8080.
  // Otherwise (through Nginx or production) → relative /api works same-origin.
  private apiUrl = window.location.port === '4200'
    ? 'http://localhost:8080/api'
    : '/api';

  constructor(private http: HttpClient) {}

  // Called by DashboardComponent on page load.
  getPlayers(): Observable<any> {
    return this.http.get(`${this.apiUrl}/players.php`);
  }

  // Called by PlayerDetailComponent when a row is clicked.
  getPlayer(id: number): Observable<any> {
    return this.http.get(`${this.apiUrl}/player.php?id=${id}`);
  }
}

Angular 17's ng generate creates components without standalone: true by default. You must add it manually. export class DashboardComponent implements OnInit {
  players: any[] = [];
  selectedPlayerId: number | null = null;

  constructor(private playerService: PlayerService) {}

  ngOnInit() { this.loadPlayers(); }

  loadPlayers() {
    this.playerService.getPlayers().subscribe({
      next: (data) => {
        this.players = data.sort((a: any, b: any) =>
          (b.flagged ? 1 : 0) - (a.flagged ? 1 : 0)
        );
      },
      error: (err) => console.error('Failed to load players:', err)
    });
  }

  selectPlayer(id: number) { this.selectedPlayerId = id; }
  closeDetail() { this.selectedPlayerId = null; }
}

import { Component, Input, Output, EventEmitter, OnChanges } from '@angular/core';
import { CommonModule } from '@angular/common';

@Component({
  selector: 'app-player-detail',
  standalone: true,              // required
  imports: [CommonModule],       // needed for *ngIf and *ngFor in template
  templateUrl: './player-detail.component.html',
  styleUrls: ['./player-detail.component.css']
})
export class PlayerDetailComponent implements OnChanges {
  @Input() playerId!: number;
  @Output() close = new EventEmitter<void>();
  data: any = null;

  constructor(private playerService: PlayerService) {}

  // OnChanges fires whenever @Input() changes — handles both first mount
  // and switching to a different player while the popup is already open.
  ngOnChanges() {
    if (this.playerId) {
      this.playerService.getPlayer(this.playerId).subscribe({
        next: (result) => this.data = result,
        error: (err) => console.error('Failed to load player detail:', err)
      });
    }
  }
}

25 nginx/render.conf

Key differences from default.conf

listen ${PORT} is a template — start.sh replaces it with Render's actual port via envsubst. fastcgi_pass 127.0.0.1:9000 (localhost) instead of app:9000 — Nginx and PHP-FPM are in the same container now. The try_files ... /index.html fallback makes Angular's client-side routing work.

server {
    listen ${PORT} default_server;
    root /var/www/html;
    index index.html;

    location /api/ {
        root /var/www;
        try_files $uri =404;
        fastcgi_pass 127.0.0.1:9000;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
26 supervisord.conf

Why this is needed

Docker expects one process per container. Locally, Nginx and PHP-FPM are separate containers. In production they're in the same container, so supervisord runs both, restarts either if it crashes, and forwards logs to stdout/stderr so Render's log viewer shows them. nodaemon=true keeps supervisord in the foreground — Docker requires this.

[supervisord]
nodaemon=true
logfile=/dev/null
logfile_maxbytes=0

[program:php-fpm]
command=/usr/local/sbin/php-fpm --nodaemonize
autostart=true
autorestart=true
stdout_logfile=/dev/stdout
stdout_logfile_maxbytes=0
stderr_logfile=/dev/stderr
stderr_logfile_maxbytes=0

[program:nginx]
command=/usr/sbin/nginx -g "daemon off;"
autostart=true
autorestart=true
stdout_logfile=/dev/stdout
stdout_logfile_maxbytes=0
stderr_logfile=/dev/stderr
stderr_logfile_maxbytes=0
27 start.sh

Injects Render's dynamic $PORT

Render assigns a random port at runtime. Nginx config is static and can't read env vars directly. envsubst replaces ${PORT} in the template with the actual value, writes the real config, then exec supervisord starts both processes.

#!/bin/sh
PORT=${PORT:-10000}

echo "Starting with PORT=$PORT"

envsubst '${PORT}' < /etc/nginx/conf.d/render.conf.template > /etc/nginx/conf.d/default.conf

rm /etc/nginx/conf.d/render.conf.template

exec /usr/bin/supervisord -c /etc/supervisor/conf.d/supervisord.conf

Lessons learned from the real build

1

Docker aborts the entire build if any Dockerfile is empty

Even if frontend/ isn't ready, a one-line placeholder keeps all other containers running. This is why Step 7b exists as a separate step now.

2

<?php must be on line 1 — nothing before it

Even a comment before <?php causes PHP to serve the file as plain text. Angular receives the comment text, fails to parse it as JSON, and gives a confusing SyntaxError that doesn't mention PHP at all.

3

Angular 17 defaults to standalone — not NgModule

Every component needs standalone: true and CommonModule in its imports for *ngFor and *ngIf to work. provideHttpClient() goes in app.config.ts, not app.module.ts.

4

Global styles.css must set the dark background

Component CSS is scoped. The browser default is white. Without body { background: #0d1117 } in styles.css, the dark-on-dark component styles render as dark text on white — nearly unreadable.

5

The 502 shows "Up" before Angular finishes building

The container starts and Docker marks it running, but Angular's first build inside the container takes ~60 seconds. A 502 right after make up usually just means wait — check docker compose logs frontend to confirm.
